# BM 导入模板字段映射

本 skill 默认对齐 `BM测试用例库_基础用例_导入模板-先锋.xlsx` 的 `template` sheet。

## 导入列

| 导入列 | 是否必填 | 填写规则 | 本 skill 映射口径 |
|------|------|------|------|
| 标题* | 是 | 填写单条用例名称，需可读、可执行、可区分 | 用“条件 + 动作 + 期望”的短句命名，不写无意义编号 |
| 所属分组 | 否 | 需填写当前用例库已存在的完整分组路径，层级之间用 `|` 分隔 | 按“玩法/系统|模块|子功能”组织，例如 `先锋组合活动|开启条件` |
| 维护人 | 否 | 必须是企业内已有成员名；错误会导致导入失败 | 未提供时留空 |
| 前置条件 | 否 | 填写执行前状态、账号条件、资源条件、时间条件 | 精简但要可执行 |
| 步骤描述 | 否 | 每条“操作步骤-预期结果”写一行，多步可换行或加 `【1】` 编号 | 多步骤统一使用 `【1】`、`【2】`、`【3】` |
| 预期结果 | 否 | 与步骤描述逐行对应 | 结果要可观察、可判定 |
| 用例等级 | 否 | 必须匹配已有值；模板示例使用 `P1` | 默认使用 `P0 / P1 / P2 / P3` |
| 用例类型 | 否 | 必须匹配系统内已有值；填错会导入失败 | 未知枚举时留空 |

## 输出约束

- 默认直接输出可导入表格，不额外插入会破坏导入的列。
- 如果需要补充“需求依据”“假设”“自动化建议”，放在表格外单独列出。
- `所属分组` 必须尽量贴近用户已有分组口径；如果用户已给出分组树，优先复用原值。
- `步骤描述` 和 `预期结果` 必须对齐。单步用例可各写一行，多步用例按编号逐行对应。
- 不要在单元格中写“待补充”“未知维护人”这类占位值；不能确认时宁可留空。

## 推荐命名模式

- 开启条件类：`创建角色后，首日达到4级，查看活动是否开启`
- 展示校验类：`7日后查看活动入口是否展示`
- 领奖幂等类：`达标后连续点击领取奖励`
- 并发协作类：`两名成员同时加入同一集结`

## 默认分组拆解建议

- 活动入口与显示：`活动名|开启条件`
- 玩法参与规则：`活动名|参与条件`
- 积分与进度：`活动名|积分/进度`
- 奖励与领取：`活动名|奖励领取`
- 排行与结算：`活动名|排行榜/结算`
- 联盟协作：`活动名|联盟协作`

## 组合活动分组建议

当需求是“组合活动 / 多页签活动 / 主入口下挂多个子玩法”时，优先按以下层级拆分：

- 组合入口层：`组合活动名|入口与页签|子功能`
- 子活动层：`子活动名|模块|子功能`
- 通用复用层：`子活动名|通用复用|子功能`
- 结束与补发层：`活动名|活动结束/补发|子功能`

示例：

- `派对组合活动|入口与页签|红点聚合`
- `派对射击|自动射击|资源不足`
- `派对拼图|宝箱奖励|大宝箱`
- `派对BP|红点与补发|结束补发`

如果用户已经给出明确分组名，以用户分组为准。
