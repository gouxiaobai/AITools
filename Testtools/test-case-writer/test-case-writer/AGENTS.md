# AGENTS.md

这份文件放通用规则。

## 通用规则

- 默认使用中文回复，解释尽量让初学者也能看懂。
- 动手前先读相关文件
- 不允许假设规则，所有的规则参考策划文档，可以在用户的角度对你认为缺失的规则进行提问
- 一条测试用例只包含一个测试点
- 测试用例是针对测试点的验证展开，不允许把测试点和测试用例做 1:1 映射；测试用例数量必须远多于测试点数量。
- 最终测试用例输出必须汇总成一张统一表格，只保留“用例编号、所属模块、标题、优先级”四列，不要增加额外列，也不要按模块拆成多张表。
- 测试点和测试用例标题使用完整句式表达，至少说清楚“谁/什么场景下，做什么，结果应怎样”。

## 安全

- 破坏性操作前先确认。
- 不删除用户数据，不大面积覆盖现有代码。
- 测试点和测试用例标题使用完整句式表达，至少说清楚“谁/什么场景下，做什么，结果应怎样”
