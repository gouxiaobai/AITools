# Test Case Writer

先遵循仓库里的 `AGENTS.md`。


## 项目背景

LAS 是一款末日生存题材的 SLG 手游（全称 Last Age: Survival），核心玩法包括基地建设、英雄养成、联盟社交、跨服对战等。活动设计需要契合末日生存世界观，同时服务于商业化目标。

已有的活动类型参考：先锋冲榜、军备竞赛、战士通行证、联盟对决、转盘、密码箱、派对组合、充值活动等。

## 目标

为当前功能或问题写出清晰、可维护、覆盖重点场景的测试用例。

## 核心原则

- 测试点定义“需要验证什么”，测试用例定义“如何验证这些测试点”。
- 测试用例是针对测试点的验证展开，不允许把测试点直接改写成 1:1 的测试用例。
- 因此测试用例数量必须远多于测试点数量；如果两者数量接近，说明用例展开不充分，需要继续补足正向、逆向、边界、状态变化和结果反馈等验证场景。
- 最终测试用例输出必须使用一张统一表格，只保留“用例编号、所属模块、标题、优先级”四列。
- 不要在最终测试用例表中增加“对应测试点”“用例类型”等额外列，也不要按模块拆成多张表。

## 工作流程

1. 先调用 `references/feature-breakdown.md` 理解功能，返回功能结构。
2. 基于测试点分析展开测试用例时，先检查每个测试点是否已经拆出足够的验证场景，避免生成接近 1:1 的测试点映射用例。
3. 用例输出保存到当前目录，保存为md文件，文件名用策划案的名称命名
