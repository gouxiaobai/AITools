# 故障排查手册

## 常见错误码
- `PARAM_RECOMMEND_FAILED`：参数推荐失败。
- `PARAM_DIFF_FAILED`：差异预检失败。
- `PARAM_APPLY_CONFLICT`：参数版本冲突。
- `PARAM_APPLY_FAILED`：参数应用失败（非冲突类）。
- `PARAM_ROLLBACK_FAILED`：参数回滚失败。

## 快速定位步骤
1. 执行 `param-monitor --days 7` 查看失败分布与慢任务。
2. 按 `run_id / proposal_id / apply_log_id` 关联日志。
3. 先判断是数据问题、版本冲突还是闸门拦截。
4. 必要时用 `param-rollback` 快速恢复。

## 回滚策略
- 单次异常：回滚最近 `apply_log_id`。
- 持续劣化：执行 `param-risk-guard` 自动回滚候选。
- 回滚后：复盘实验参数与成本假设是否失真。
